1. Update Prayer Times for a New Month

{% if mosque.parserFormat == 'tralee' %}

Prayer times come from the Imam as an Excel spreadsheet. Open the file, click the tab for the month (e.g. JUNE), and follow the steps below.

Expected spreadsheet layout

The parser expects exactly this structure:

JUNE		ISLAMIC 	BEGINNING TIMES...
DATE	DAY	HIJRI	FAJR	SUNRISE	ZOHR	ASAR	MAGRIB	ISHA		FAJR	ZOHR	ASAR	MAGRIB	ISHA
1	Mon	15 D.Hijjah	03:04	05:23	13:37	19:12	21:53	23:13		4.45	2.00	8.15	9.58	11.25
2	Tue	16	03:04	05:22	13:37	19:12	21:54	23:14		"	"	"	9.59	"
3	Wed	17	03:04	05:22	13:37	19:13	21:55	23:14		"	"	"	10.00	"
...
30	Sun	15 Muharram	03:46	05:54	13:34	19:02	22:18	23:43		"	"	"	10.00	"
Key rules:
  • Row 1 — the month name (JUNE) must be in the first cell; the parser uses it to identify the month.
  • Row 2 — column headers; must be present.
  • " (quote symbol) — means "same Jamaah time as the row above"; the parser fills these in automatically.
  • Decimal afternoon times — Jamaah times are written as H.MM decimal: 4.45 → 16:45  |  2.00 → 14:00  |  8.15 → 20:15  |  9.58 → 21:58
  • Early morning times (e.g. Fajr Jamaah 4.45 before sunrise) are kept as 04:45.
{% else %}

Prayer times come as a tab-separated table. The parser auto-calculates Iqamah as Adhan + 10 minutes for every prayer.

Expected table layout

Day	Date	Fajr	Sunrise	Dhuhr	Asr	Maghrib	Isha	Iqama + 10 min
Wed	1	5:19	6:58	13:31	17:02	20:01	21:33	
Thu	2	5:17	6:55	13:31	17:03	20:03	21:35	
Fri	3	5:14	6:53	13:31	17:04	20:04	21:37	
…								
Thu	30	4:05	5:51	13:24	17:27	20:53	22:32	
...
Key rules:
  • No month name row needed — select the month from the dropdown before parsing.
  • Times use 24-hour HH:MM format. No decimal times.
  • Iqamah = Adhan + 10 min, calculated automatically for every prayer.
{% endif %}

Steps

1
{% if mosque.parserFormat == 'tralee' %} In Excel, click on the cell containing the month name (JUNE), then select down to the last day and across to the last Jamaah column. Copy (Ctrl+C). {% else %} Copy the month's data from the source spreadsheet — include the header row down through the last day. Copy (Ctrl+C). {% endif %}
2
Open Prayer Times and click + Add Month to open the upload panel. {% if mosque.parserFormat == 'dublin' %}Select the month from the dropdown.{% endif %} Paste (Ctrl+V) the copied data into the text area.
3
Click Parse & Save to GitHub. This parses the pasted data and commits it in one step. A green toast confirms success. The live display picks up the new times within a few minutes.

2. Update Prayer Time for a Specific Day or Fix a Wrong Prayer Time

1
Open Prayer Times and find the row for the day to edit.
2
Click the ✎ (pencil) button on the row — this opens the Edit Prayer Times modal.
3
Update the time(s) in the modal. Format: HH:MM (e.g. 19:45).
4
Click Save & Commit in the modal footer — saves and commits to GitHub in one step.

3. Add a Text Announcement

Use text announcements for notices like: "Eid prayer: Saturday 7:00am — arrive early" or "Weekly lecture: Every Tuesday after Isha."

1
Open Announcements and click + Text to expand the Add Text panel.
2
Fill in the form:
FieldNotes
IDA unique identifier with no spaces, e.g. eid_dinner_2027.
MessageThe text shown on the display screen.
Schedule WindowsRequired. Set Start date/time → End date/time for when the announcement is active. Click + Add Window to add more (max 5).
Special announcementCheck this for high-priority notices.
3
Click Save to GitHub.

5. Cancel a Recurring Event Temporarily

If a regular class (e.g. Quran class every Tuesday) is cancelled for specific dates, use a Suppress Period instead of deleting the announcement.

Note: Suppress periods are only available on recurring and control type announcements.
1
Open Announcements and find the recurring announcement.
2
Click Edit.
3
In the edit modal, scroll to Suppress periods → click + Add Suppress Period.
4
Set the Start date/time and End date/time of the cancelled window.
5
Click Save to GitHub in the modal footer.

The announcement disappears during the suppress window, then resumes automatically. The Suppressed now badge shows while it is suppressed.

6. Hide an Announcement Immediately

To remove an announcement from the display without deleting it (so you can restore it later):

1
In the Announcements list, click the Hide button on the row — this commits to GitHub immediately, no separate save step needed.

The announcement stays in your list with a Hidden badge. To restore it: click the Show button (same button, label toggles to "Show" while hidden) — also saves immediately.

9. Adjust Adhkar Poster Timings

{% else %}

8. Adjust Adhkar Poster Timings

{% endif %}

Adhkar slides automatically appear after each prayer. Settings has a per-prayer table and a separate Friday Jumu'ah table.

1
Open Settings and scroll to Adhkar Display Timing.
2
In the Per-Prayer Settings table, edit any prayer row: Delay after Jamaah (min) — pause before Adhkar starts; Display window (min) — total minutes the cycle is active; Poster 1 (sec) — Poster 2 fills remaining window time automatically.
3
For Friday Jumu'ah Adhkar, edit the Friday Dhuhr table: set Summer BST trigger and Winter GMT trigger (clock times when Adhkar starts on Fridays).
4
Click Save to GitHub.

11. GitHub Token Setup

{% else %}

10. GitHub Token Setup

{% endif %}

All saves go through the GitHub API so changes are versioned and auto-deployed to the live site.

Create a token (one-time)

1
Go to GitHub → Settings → Developer settings → Fine-grained personal access tokens → Generate new token.
2
Set Repository access to this repo only.
3
Under Permissions → Repository permissions → Contents, select Read and write.
4
Click Generate, then copy the token immediately — you can only see it once.

Save the token in your browser

1
Open Dashboard.
2
Paste the token into the GitHub Token field and click Save.

The token is stored only in your browser's localStorage. It is never sent to the server. You only need to do this once per browser (and again in a different browser or private/incognito mode).
